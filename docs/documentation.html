1. Project Overview

Birthday Song Lookup is a web application that, given a date of birth, returns:

  • The day of the week for that date
  • The #1 song on the USA Billboard Hot 100 chart
  • The #1 song on the UK Singles Chart
  • The #1 song on the Yugoslavia/Croatia regional chart
  • World events that happened on that exact date (same year)
  • Notable events that happened on that same day throughout history

Key Features

FeatureDescription
Multi-chart lookupUSA, UK, and regional (Yugoslavia/Croatia) charts
Date validationChart availability checks (Billboard: Aug 4, 1958; UK: Nov 14, 1952)
World EventsHistorical events from Wikipedia on the exact date
Events This DayNotable events on the same day across all years
Tabbed UIClean tabbed interface: Songs, Events on Date, Events This Day
ResponsiveWorks on mobile and desktop browsers

2. Architecture

High-Level Architecture

+------------------+ +-------------------+ +----------------------+ | | HTTP | | HTTPS | | | Browser Client | ------> | Express Server | ------> | External APIs | | (index.html) | <------ | (server.ts) | <------ | | | | JSON | Port 3000 | JSON | - Billboard (GitHub)| +------------------+ +-------------------+ | - UK Official Charts| | - Wikipedia REST API| +----------------------+

Request Flow

  1. User enters date in browser form
  2. Frontend sends POST /api/lookup with JSON body {"date": "YYYY-MM-DD"}
  3. Server validates date format
  4. Server makes parallel requests to:
    • Billboard Hot 100 data (GitHub JSON)
    • UK Official Charts (web scraping)
    • Wikipedia On This Day API (2 calls: on-date + this-day)
  5. Server computes regional chart from local data
  6. Server returns combined JSON response
  7. Frontend renders results in tabbed UI

Concurrency Model

All external API calls are made in parallel using Promise.all() to minimize response time:

const [usaChart, ukChart, worldEventsOnDate, worldEventsThisDay] = await Promise.all([
  getUSAChart(date), getUKChart(date), getWorldEventsOnDate(date), getWorldEventsThisDay(date)
]);

Why These Choices?

  • Express.js - Minimal, well-known, easy to deploy
  • TypeScript - Type safety for complex API responses
  • tsx - Run TypeScript directly without build step (dev mode)
  • Vanilla frontend - No framework needed for this simple UI, fast loading
  • Native https - No extra dependencies for HTTP calls

7. Installation & Running

Prerequisites

  • Node.js 18+ (recommended: 22)
  • npm

Setup

git clone https://github.com/emarzig/birthday-song-lookup.git
cd birthday-song-lookup
npm install

Run (Development)

npm run dev

Opens server at http://localhost:3000

Run (GitHub Codespace)

npm run dev

Server binds to 0.0.0.0:3000. Use the Codespace port-forwarded URL.

Known Limitations

  • Regional chart data is hardcoded with only a few sample entries
  • UK chart scraping depends on Official Charts website HTML structure
  • Wikipedia API may not have events for every date/year combination
  • Billboard data depends on third-party GitHub repository availability
  • No caching - every request fetches fresh data from external APIs
  • No authentication - API is open to anyone who can reach the server

Error Handling

Each data source has independent error handling. If one API fails, the others still return data. Users see specific "not available" messages per data source.
